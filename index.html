Technical Overview

This prototype demonstrates a Human-in-the-Loop (HITL) workflow for managing transport data via natural language. It leverages Generative AI to orchestrate read/write operations securely.

1. Current Architecture (POC)

The system operates on a serverless architecture optimized for rapid prototyping:

  • AI Orchestrator: Google Vertex AI Agent Builder powered by Gemini 1.5 Flash for low-latency reasoning.
  • Middleware: A Google Apps Script deployed as a Web App acts as the API Gateway. It parses JSON payloads to determine intent (search vs update).
  • Database & UI: Google Sheets serves as the mutable database. The script performs real-time DOM manipulation (highlighting updated cells in yellow) to provide immediate visual feedback in the frontend.

2. Workflow Logic & Security

The agent does not hallucinate SQL. It follows a strict Function Calling protocol defined via OpenAPI 3.0:

  • Intent Detection: The LLM extracts parameters (Carrier, Origin, Destination, Percentage) from user input.
  • Safety Layer: System instructions enforce a strict validation loop. The update action is cryptographically blocked until the user explicitly confirms the simulation results.

3. Production Target Architecture

For the production environment, the stack will migrate to ensure scalability and ACID compliance:

  • Database: Migration from Sheets to PostgreSQL (Cloud SQL) to handle concurrent writes and large datasets.
  • Backend: Replacement of Apps Script with a dedicated Python (FastAPI) service.
  • Auditability: Implementation of a dedicated audit_log table to track every AI-initiated change for compliance.
